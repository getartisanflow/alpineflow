# v0.2.1-alpha Migration Guide

**Focus:** Builder-oriented improvements to how AlpineFlow measures and lays out nodes. Additive feature surface; a handful of alpha-acceptable behavior changes documented below.

## New `FlowNode` properties

All optional; existing node definitions continue to work unchanged.

- `fixedDimensions?: boolean` â€” opt-in to inline `style.height` on leaf nodes AND opt out of ResizeObserver updates. Use for decorative or fixed-size nodes. **Auto-promoted when the system drives a height**: resize drag, compute output, and animation of `dimensions.height` all set this to `true` on the affected node.
- `resizeObserver?: boolean` â€” default `true`. Set `false` to exclude a node from the shared ResizeObserver (e.g., annotation nodes or decorative overlays where measurement is noise).
- `minDimensions?: Partial<Dimensions>` â€” lower bound for observed dimensions. Either `width` or `height` may be omitted ("no lower bound on that axis"). Applied by the ResizeObserver before updating `node.dimensions`.
- `maxDimensions?: Partial<Dimensions>` â€” upper bound. Use `Infinity` for unbounded.

## New canvas method

### `canvas.batch(fn)`

Suspend layout reconciliation during `fn`, then run a single reconciliation pass after. Ref-counted (safe to nest â€” inner batches join the outer). Returns `fn`'s return value. Reconciles on throw via `try/finally` so a throwing `fn` still reconciles before the error propagates.

```js
// Adding 100 nodes that share a parent would previously trigger 100 layouts.
// With batch(), exactly one layout runs per affected parent.
$flow.batch(() => {
    $flow.addNodes(allNodes);
    $flow.addEdges(allEdges);
});
```

Access as `canvas.batch(...)` in static code, or `$flow.batch(...)` from Alpine scope inside `<x-flow>` (WireFlow) or `x-data="flowCanvas(...)"` elements.

## Behavior changes

### Leaf nodes no longer get inline `height` (A2)

Before v0.2.1-alpha, every node received `style.height = dimensions.height + 'px'`. Leaf nodes with rich templated content (Alpine `x-if` / `x-for` templates, form controls) had their height locked to the initial (pre-render) measurement, clipping content that grew after templates populated.

**Now:** only nodes with `childLayout` (containers) or `fixedDimensions: true` receive inline height. Leaf nodes let their content determine height; the new ResizeObserver captures the natural rendered height back into `node.dimensions` for parent layout calculations.

**Migration:** if you were setting explicit height on leaf nodes and expected it to persist, add `fixedDimensions: true` to those nodes. Otherwise the value will be overwritten by measurement.

**Auto-promotion:** three system paths automatically set `fixedDimensions = true` on affected nodes:

- Resize drag â€” user dragged a resize handle. Node now has an explicit height.
- Compute output â€” if `compute()` writes a height, the node is authoritative-sized.
- Animation â€” if an animation target includes `dimensions.height`, the node is promoted at animation start and stays fixed.

### Layout lifecycle now reactive (A1, A3, A4)

- **A1 (ResizeObserver):** a single shared `ResizeObserver` tracks every node element. When rendered dimensions change, `node.dimensions` updates automatically and the parent re-lays out. No more stale measurements.
- **A3 (reactive childLayout):** mutating `node.childLayout.columns` (or `gap`, `padding`, `headerHeight`, `direction`, `stretch`) triggers re-layout automatically. No need to call `layoutChildren(parentId)` manually.
- **A4 (addNodes auto-layouts parents):** `addNodes` now re-lays out affected parent containers, consistent with `removeNodes` behavior.

**Migration:** if your code called `canvas.layoutChildren(parentId)` manually after mutations, you can remove those calls. Keep them only if they cover a path the new reactivity doesn't (rare).

### Layout dedup â€” at most one `layoutChildren` per parent per frame

Multiple triggers in the same frame (e.g., A4 on add + A1 on observer callback) collapse to a single `layoutChildren` call per parent per animation frame. Frame-aligned, not time-based. Includes a cross-frame loop safety net: a parent laid out in >5 consecutive frames is suppressed with a `console.warn` until the next user mutation (`addNodes`, `removeNodes`) clears the counter.

**Migration:** if tests or tooling count `layoutChildren` calls and depend on the old duplicate-call behavior, expect lower counts. Test assertions that expected 2Ă— layout per mutation (once for A4, once for A1) should now expect 1.

### Dimensions are client-local, never synced

`node.dimensions` now updates reactively from the ResizeObserver. Different clients may measure different values (fonts, browser zoom, DPI). Dimensions are **excluded from collab sync** â€” they're local state, not part of the canonical document. This only matters if you're using Yjs collaboration.

## Not yet reachable (planned)

- `patchConfig` does not accept a `nodes` key â€” it's canvas-level config only. Server-side node/childLayout mutation uses direct reactive assignment (via the wire-bridge `flow:updateNode` command or `$wire.entangle`).
- `runState` (D2), controlled-mode (D1), and the animate/wire-bridge extractions are planned for v0.3.0-alpha. See `project_alpineflow_animate_extraction` memory note.

## Installation / upgrade

```bash

npm run build
```

For WireFlow consumers, a follow-up WireFlow bump pulls the new dist â€” no code change if you accept the default `wireflow:install` output.
